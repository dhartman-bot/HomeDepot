Making Your Kickoff Successful

To ensure a productive pilot kickoff session with 50+ developers, complete these preparation steps:

Before the Kickoff

‚ö†Ô∏è CRITICAL: Send Invitations FIRST

Before your team starts ANY setup steps, ensure ALL participants have:

  • Received organization invitation emails
  • Accepted the invitation and created their Claude accounts
  • Verified they can log in to claude.ai and see your organization name

üö® Common failure mode: Users install Claude Code BEFORE accepting invites, then see a billing page and get confused.

1. Everyone Completes Setup (Steps 1-5 Below)

  • Send invitations at least 48 hours before the kickoff
  • Have all participants complete the full installation and setup before the kickoff meeting
  • This ensures the kickoff focuses on usage, not troubleshooting installations
  • Target: 100% of participants have Claude Code working by kickoff time

2. Identify Your Champions

  • For a group of 50+ developers, identify 5-10 champions already familiar with Claude Code
  • Aim for 1 champion per 5-10 participants for effective support
  • Champions will provide peer support during and after the kickoff
  • Champions can help troubleshoot issues while the session continues
  • Consider champions who are enthusiastic about AI-assisted development and strong communicators

3. Select Key Use Cases for Live Demos

Choose 2-3 real use cases from your actual work to demonstrate during kickoff:

  • "Debug a production issue" - Show how to analyze stack traces and find root causes
  • "Onboard to unfamiliar code" - Navigate and understand a complex module
  • "Generate unit tests" - Create comprehensive test coverage for existing code
  • "Refactor legacy code" - Modernize old code while maintaining behavior
  • "Review code before PR" - Get AI feedback on quality and potential issues

During the Kickoff

Suggested Agenda (60-minute session):

  • 5 min: Quick installation verification (everyone confirm Claude Code works)
  • 10 min: Review pilot goals and success metrics
  • 30 min: Live demos of 2-3 key use cases with your actual codebase
  • 10 min: Q&A and troubleshooting
  • 5 min: Week 1 action items and champion introductions

5.5 Customize Claude Code for Your Workflow (Optional but Recommended)

Before diving into daily use, take 5 minutes to customize Claude Code for your projects. This will make Claude much more helpful and context-aware.

Create a CLAUDE.md File for Your Project

A CLAUDE.md file is like a README for Claude. It tells Claude Code about your project's context, coding standards, and team preferences. Place it in your project root directory.

Why this matters: Claude will automatically read this file when working in your project, giving it essential context about your codebase, conventions, and goals.

Example CLAUDE.md for a typical Home Depot project:

# Project Context ## Overview This is [Project Name], a [web application/microservice/etc.] that handles [brief description]. Built with [tech stack: Java/Spring Boot, React, etc.]. ## Coding Standards - Follow Home Depot's enterprise Java coding guidelines - Use meaningful variable names (no abbreviations unless standard) - All new code must include unit tests (JUnit 5 + Mockito) - API endpoints follow RESTful conventions - Database queries use JPA/Hibernate (no raw SQL unless justified) ## Architecture - Microservices architecture with Spring Boot - PostgreSQL database - Redis for caching - Kafka for event streaming - Deployed on Azure Kubernetes Service (AKS) ## Key Directories - `/src/main/java` - Java source code - `/src/test/java` - Unit and integration tests - `/src/main/resources` - Configuration files - `/docs` - Architecture and API documentation ## Dependencies - Spring Boot 3.x - Java 17 - Maven for build management ## Testing Requirements - Minimum 80% code coverage for new code - All PRs must pass CI/CD pipeline - Integration tests for all API endpoints ## Common Tasks - Run tests: `mvn test` - Run locally: `mvn spring-boot:run` - Build: `mvn clean package` ## Security Considerations - Never commit credentials or API keys - Use Azure Key Vault for secrets - All endpoints require authentication (OAuth 2.0) - PII data must be handled per GDPR/CCPA requirements ## Notes for Claude - When generating code, prefer functional programming patterns - Include JavaDoc comments for public methods - Suggest performance optimizations when working with database queries - Flag potential security issues proactively

How to create your CLAUDE.md:

cd /path/to/your/project
claude "Create a CLAUDE.md file for this project"

Claude will analyze your project and draft a custom CLAUDE.md file for you!

Network Connectivity Issues - Proxy, VPN, or Firewall

üåê COMMON IN ENTERPRISE: Corporate networks often block or intercept connections to external APIs like Anthropic's services.

Symptoms of network blocking:

  • Connection timeouts when running claude
  • SSL/TLS certificate errors (SELF_SIGNED_CERT_IN_CHAIN, UNABLE_TO_VERIFY_LEAF_SIGNATURE)
  • "Unable to connect to Anthropic services" errors
  • Authentication fails despite correct credentials
  • Works on home network but not corporate network

Common causes:

  • Corporate proxy performing SSL/TLS inspection (man-in-the-middle)
  • Firewall rules blocking outbound connections to *.anthropic.com or *.claude.ai
  • VPN restrictions preventing API access
  • Network security software intercepting HTTPS traffic
  • Self-signed certificates in your organization's certificate chain

‚úÖ Recommended Solutions (Contact Your IT Department)

Work with your IT/Network team to implement these proper solutions:

  1. Whitelist Anthropic domains:
    • *.anthropic.com
    • *.claude.ai
    • api.anthropic.com (specifically)
  2. Exclude from SSL/TLS inspection: Configure your proxy to bypass SSL inspection for Anthropic domains (allows direct encrypted connection)
  3. Add root CA certificate to trust store: If SSL inspection is required, have IT add your organization's root CA certificate to your system's trusted certificate store
  4. Configure proxy settings: Set HTTP_PROXY and HTTPS_PROXY environment variables if needed:

    macOS/Linux (add to ~/.zshrc or ~/.bashrc):

    export HTTP_PROXY="http://proxy.company.com:8080"
    export HTTPS_PROXY="http://proxy.company.com:8080"

    Windows PowerShell (add to $PROFILE):

    # Set for current session:
    $env:HTTP_PROXY="http://proxy.company.com:8080"
    $env:HTTPS_PROXY="http://proxy.company.com:8080"

    # Or set permanently (run as Administrator):
    [System.Environment]::SetEnvironmentVariable("HTTP_PROXY", "http://proxy.company.com:8080", "User")
    [System.Environment]::SetEnvironmentVariable("HTTPS_PROXY", "http://proxy.company.com:8080", "User")

‚ö†Ô∏è Do NOT Use Insecure Workarounds

Avoid disabling SSL certificate verification (e.g., NODE_TLS_REJECT_UNAUTHORIZED=0). This:

  • Makes your connection vulnerable to man-in-the-middle attacks
  • Violates security best practices
  • May violate your organization's security policies
  • Doesn't actually fix the root cause

Always work with IT to implement proper solutions above.

Quick diagnostic steps:

  1. Test basic connectivity:

    macOS/Linux (Terminal):

    curl -I https://api.anthropic.com/v1/messages

    Windows (PowerShell):

    Invoke-WebRequest -Uri "https://api.anthropic.com/v1/messages" -Method Head

    You should see HTTP response headers. If you get connection refused or timeout, network is blocking.

  2. Check if proxy is in use:

    macOS/Linux (Terminal):

    echo $HTTP_PROXY
    echo $HTTPS_PROXY

    Windows (PowerShell):

    $env:HTTP_PROXY
    $env:HTTPS_PROXY
    # Also check system proxy settings:
    netsh winhttp show proxy
  3. Test from different network: Try from home WiFi or mobile hotspot to confirm if it's enterprise network-specific
  4. Check certificate errors:

    macOS/Linux (Terminal):

    openssl s_client -connect api.anthropic.com:443 -servername api.anthropic.com

    Windows (PowerShell):

    # Test TLS connection and view certificate
    $url = "https://api.anthropic.com"
    $req = [System.Net.HttpWebRequest]::Create($url)
    try {
        $req.GetResponse() | Out-Null
    } catch {
        Write-Host "Error: $_" -ForegroundColor Red
    }
    $req.ServicePoint.Certificate | Format-List *

    Look for certificate chain issues or self-signed certificates in the output.

üí° Business justification for IT: Claude Code is an approved enterprise tool requiring API access to *.anthropic.com for your organization's pilot program. Whitelisting these domains enables developer productivity improvements while maintaining security through proper certificate validation.
