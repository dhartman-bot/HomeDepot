Making Your Kickoff Successful

To ensure a productive pilot kickoff session with 50+ developers, complete these preparation steps:

Before the Kickoff

1. Everyone Installs Claude Code (Complete Steps 1-5 Below)

  • Have all participants complete the installation and setup before the kickoff meeting
  • This ensures the kickoff focuses on usage, not troubleshooting installations
  • Target: 100% of participants have Claude Code working by kickoff time

2. Identify Your Champions

  • For a group of 50+ developers, identify 5-10 champions already familiar with Claude Code
  • Aim for 1 champion per 5-10 participants for effective support
  • Champions will provide peer support during and after the kickoff
  • Champions can help troubleshoot issues while the session continues
  • Consider champions who are enthusiastic about AI-assisted development and strong communicators

3. Select Key Use Cases for Live Demos

Choose 2-3 real use cases from your actual work to demonstrate during kickoff:

  • "Debug a production issue" - Show how to analyze stack traces and find root causes
  • "Onboard to unfamiliar code" - Navigate and understand a complex module
  • "Generate unit tests" - Create comprehensive test coverage for existing code
  • "Refactor legacy code" - Modernize old code while maintaining behavior
  • "Review code before PR" - Get AI feedback on quality and potential issues

During the Kickoff

Suggested Agenda (60-minute session):

  • 5 min: Quick installation verification (everyone confirm Claude Code works)
  • 10 min: Review pilot goals and success metrics
  • 30 min: Live demos of 2-3 key use cases with your actual codebase
  • 10 min: Q&A and troubleshooting
  • 5 min: Week 1 action items and champion introductions
üí° Pro Tip: Record the kickoff session so participants can reference the demos later.

3 Install Claude Code

macOS Installation

Option 1: Using Homebrew (Recommended)

brew install --cask claude-code

Option 2: Using curl

curl -fsSL https://claude.ai/install.sh | bash

Linux Installation

Using curl:

curl -fsSL https://claude.ai/install.sh | bash

Alternative (Homebrew on Linux):

brew install --cask claude-code

Windows Installation

Option 1: PowerShell (Recommended - right-click and "Run as Administrator")

irm https://claude.ai/install.ps1 | iex

Option 2: Command Prompt (CMD)

curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
Note: You may need to adjust your PowerShell execution policy. If you encounter an error, run:
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
Then try the installation command again.
Windows-Specific Notes:
  • After installation, close and reopen PowerShell or Command Prompt
  • If using Windows Terminal, restart it to refresh the PATH
  • Some antivirus software may flag the installer - add an exception if needed

4 Configure Claude Code

‚ö° Recommended for Shared Console Setup: Use the environment variable method below to ensure Claude Code uses your provided API key.

Method 1: Environment Variable (Recommended)

Set the API key as an environment variable in your shell profile. This ensures Claude Code always uses the shared console API key.

For macOS/Linux (bash):

echo 'export ANTHROPIC_API_KEY="sk-ant-api03-xxxxx..."' >> ~/.bashrc
source ~/.bashrc

For macOS/Linux (zsh):

echo 'export ANTHROPIC_API_KEY="sk-ant-api03-xxxxx..."' >> ~/.zshrc
source ~/.zshrc
Note: Replace sk-ant-api03-xxxxx... with your actual API key provided by your coordinator.

For Windows PowerShell:

# Add to your PowerShell profile
notepad $PROFILE

# Add this line to the file:
$env:ANTHROPIC_API_KEY = "sk-ant-api03-xxxxx..."

For Windows Command Prompt:

setx ANTHROPIC_API_KEY "sk-ant-api03-xxxxx..."
Windows Note: After setting environment variables, close and reopen your terminal or IDE completely.

Method 2: Interactive Configuration

If you don't see the "Use API Key" option when running Claude Code, you may need to clear existing credentials first:

# Clear any existing authentication
claude logout

# Launch Claude Code
claude

When prompted, choose "Use API Key" and:

  1. Paste the API key provided by your coordinator
  2. Select Claude Sonnet 4.5 as your model

Troubleshooting: "Use API Key" Option Not Appearing

If you don't see the "Use API Key" option:
  1. Use Method 1 (Environment Variable) - This bypasses the interactive prompt entirely
  2. OR manually clear credentials:
    # macOS/Linux:
    rm -rf ~/.claude/credentials

    # Windows:
    del %USERPROFILE%\.claude\credentials
  3. Restart your terminal completely
  4. Run claude again
Model Selection: Claude Sonnet 4.5 offers the best balance of performance and capability for coding tasks, with 1M token context window.

Common Issues & Solutions

"Use API Key" Option Not Appearing

üîß COMMON ISSUE for Shared Console Setup: If Claude Code doesn't show the "Use API Key" option during setup, this is usually because it's detecting other authentication methods on your system.

Solution: Use the environment variable method (recommended):

# For macOS/Linux (bash):
echo 'export ANTHROPIC_API_KEY="your-key-here"' >> ~/.bashrc
source ~/.bashrc

# For macOS/Linux (zsh):
echo 'export ANTHROPIC_API_KEY="your-key-here"' >> ~/.zshrc
source ~/.zshrc

# Then test:
claude "hello"

Alternative: Clear existing credentials and retry:

# Clear credentials
claude logout
rm -rf ~/.claude/credentials

# Restart terminal completely, then:
claude

Authentication Failed or Invalid API Key

If you see authentication errors:

  • Verify you copied the full key including the sk-ant-api03- prefix
  • Check for extra spaces or line breaks when pasting
  • Confirm the key is still valid in the console (not revoked)
  • Try setting via environment variable instead of interactive prompt
macOS Keychain Note: If you see keychain-related errors on macOS, you may need to unlock your keychain. Claude Code securely stores credentials in your system keychain.

Rate Limit Exceeded

Contact your pilot program coordinator to increase limits. Check your usage at: console.anthropic.com/settings/usage

Claude Code Command Not Found

# Restart your terminal, then verify installation
which claude

# If not found, reinstall:
curl -fsSL https://claude.ai/install.sh | bash

Keeping Claude Code Updated

Claude Code automatically updates by default. To manually check for updates:

claude update

To disable auto-updates, set this environment variable in your shell configuration:

export DISABLE_AUTOUPDATER=1
